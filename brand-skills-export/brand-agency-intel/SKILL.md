# 品牌代理谈判情报官

一站式品牌代理评估系统。输入三个必填参数（合作目标可选），输出一份可以带着上谈判桌的报告。

## 输入参数

| 参数 | 必填 | 说明 | 示例 |
|------|------|------|------|
| **品牌名** | ✅ 必填 | 品牌全称 | earthgrow 之植 |
| **品类** | ✅ 必填 | 所属行业/赛道 | 功效护肤 / 香氛 / 宠物食品 |
| **品牌原区域** | 可选 | 品牌发源地/主力市场。不填则自动从调研中推断 | 日本东京 / 韩国首尔 / 上海 |
| **代理区域** | ✅ 必填 | 预计代理的地理范围 | 华东三省 / 全国线上 / 川渝线下 |
| **合作目标** | 可选 | 想达到的目的。不填则按「评估代理可行性」处理 | 独家代理 / 经销 / 联名合作 |

> 如果品牌原区域 = 代理区域（本土品牌做本土代理），跳过地域迁移分析。

### 数据量可信度参考

报告可信度取决于**去噪后**的有效独立结果数（已剔除品牌SEO干预）。搜索后如实记录：

| 去噪后有效结果 | 可信度 | 说明 |
|-------------|--------|------|
| < 20 条 | ⚠ 低 | 品牌太小/太新/太小众，报告仅供参考，不建议作为谈判决策依据 |
| 20-60 条 | ◐ 中 | 基础可信，有分析价值，但部分维度可能信息不足 |
| 60-150 条 | ✅ 较高 | 信息较充分，核心结论可靠 |
| > 150 条 | ✅✅ 高 | 数据充分，报告可作为谈判核心依据 |

> 输出报告时，在基本信息下方标注本次搜索的有效结果条数和可信度等级。

## 执行流程（6 步）

### Step 0：生成调研关键词

根据输入参数，自动生成 8-10 个搜索关键词，覆盖：

```
□ 品牌基础信息（企业信息、融资、团队、发源地/原区域）
□ 产品与价格（产品线、爆品、定价）
□ 口碑正负面（好评、差评、投诉）
□ 竞品对标（直接竞品、间接竞品、替代品）
□ 区域适配（品牌原区域 vs 代理区域：消费力、审美、习惯、价格接受度差异）
□ 行业趋势（品类增长、政策风险、新入局者）
```

### Step 1：自动采集公开信息

**核心策略：先打地基，再扩面，见好就收。**

#### 1a. 百度百科直取（必做，1-2分钟）

对于任何中文品牌，**第一步永远是直接导航到百度百科**。百科词条提供了结构化、可引用的品牌底盘数据（创始人、成立时间、母公司、融资、产品线概要），且受百度编辑审核机制约束，信息相对准确可比。

1. `browser_navigate` 导航到 `https://baike.baidu.com/item/{品牌名}`
2. `browser_snapshot` 获取页面内容（百科页面较长，可能需要 `full=true`）
3. 从 snapshot 中提取：品牌概要、成立时间、创始人、母公司、发展阶段、核心产品
4. 百科内容标注「📌品牌官网发布」，纳入「品牌官网内容」类别单独保留

> 如果品牌没有百度百科词条（太小/太新），直接跳转到 1b。

#### 1b. Bing 搜索扩展（5-7次搜索，逐条执行，不用子代理）

**⚠️ 严禁使用 `delegate_task` 执行搜索**。实测：每个子代理执行 5-6 次 `browser_navigate` + `browser_console` 会触发 600s 超时（34+ API调用仍无法完成）。搜索必须由主代理直接执行。

**正确的单次搜索流程**（每条搜索耗时约10-15秒）：

1. `browser_navigate` 导航到 `https://www.bing.com/search?q={URL编码关键词}&setlang=zh-cn`
2. `browser_console` 执行 JS 提取结果 —— **务必用字符串拼接而非 JSON.stringify**：

```js
// ✅ 正确：字符串拼接（不会超时）
Array.from(document.querySelectorAll('.b_algo')).slice(0,10).map(el =>
  el.querySelector('h2 a')?.textContent?.trim() + ' ||| ' +
  (el.querySelector('.b_caption p')?.textContent?.trim() || '')
).join('\n---\n')

// ❌ 错误：JSON.stringify 在大型DOM上经常超时
// JSON.stringify(Array.from(...).map(el => ({title: ..., snippet: ..., url: ...})))
```

3. 每条搜索换一次 `browser_navigate` + `browser_console`，逐条积累
4. **当发现连续2条搜索返回几乎相同的结果集时，立即停止搜索**——这意味着品牌SEO已占领所有可用关键词，继续搜只是浪费时间和重复抓取同样的官方域。标记为「SEO封锁」进入去噪阶段。

**搜索关键词清单（按优先级，不必全部执行完）：**

优先级 1 — 品牌底盘 + 产品（必做，3-4条）：
```
1. "{英文品牌名}" {中文品牌名} 品牌 创始人 母公司
2. "{英文品牌名}" {品牌名} 产品 爆品 价格 2025
3. "{英文品牌名}" 成分 技术 专利 研发
4. {品牌名} 母公司 企查查 天眼查 风险 诉讼
```

优先级 2 — 口碑 + 竞品（做 2-3 条，遇SEO封锁则停）：
```
5. "{英文品牌名}" {品牌名} 吐槽 差评 缺点 投诉
6. "{英文品牌名}" 对比 {竞品1} {竞品2}
7. {品牌名} {代理区域} 市场 渠道 门店
```

优先级 3 — 行业 + 区域（做 1-2 条，视需要）：
```
8. {品类} 行业趋势 市场规模 2025 2026
9. {代理区域} 消费差异 {品类}
```

#### 1c. 关键参数说明

> ⚠️ **中文搜索编码坑（严重）**：cn.bing.com 对纯中文品牌名存在字符级分割 bug。例如搜索「修丽可」实际返回的是单字「修」的字典释义结果——Bing 把多字品牌名拆成了独立字符。**强制规则：所有 Bing 搜索必须将英文品牌名用双引号包裹放在中文品牌名前面**（如 `"SkinCeuticals" 修丽可`），这是唯一保证 Bing 正确识别品牌的方式。仅靠中文双引号（如 `"修丽可"`）不够可靠，实测仍会降级为字符搜索。

> ⚠️ **SEO 封锁识别**：当品牌为知名品牌（尤其是有独立 .com / .cn / .hk / .tw 官网的品牌），Bing 首页结果可能 100% 被官方域占据。**识别信号**：不同关键词搜索返回相同的前 8-10 条结果（全是官网域 + 百度百科 + 电商旗舰店）。**正确应对**：立即停止搜索，不要在 10+ 条不同关键词上反复获取相同结果。接受 〈⚠低 可信度并基于百科数据和行业常识完成报告。详见 `references/bing-search-pitfalls.md`。

> 如果不知道竞品，先在搜索中确认。如果用户未提供品牌原区域，从搜索结果中推断。

> 搜索完成后，统计有效信息条数，填入报告「数据可信度」栏。

### SEO 去噪（搜索完成后、分析前执行）

从所有搜索结果中，逐条剔除品牌 SEO 主动干预，**仅保留独立第三方信息**。

**直接剔除（不计入有效结果）：**
| 污染类型 | 识别特征 | 处理 |
|---|---|---|
| 品牌官网 | 品牌官方域名（.com/.cn 等） | 剔除 |
| 品牌知识卡片 | Google/Bing 右侧品牌面板 | 剔除 |
| SEO 通稿 | 标题含「品牌故事」「品牌介绍」「正式发布」「品牌理念」等 PR 腔，来自第三方域 | 剔除 |
| SEO 农场 | 内容农场、采集站、低质目录站、明显机器生成页 | 剔除 |
| 付费种草搬运 | 小红书笔记被 SEO 站完整转载，非原创 UGC | 剔除 |

**保留但降权：**
| 类型 | 识别特征 | 处理 |
|---|---|---|
| 疑似软文 | 独立域，标题含「评测」「测评」「怎么样」「值得买吗」 | 纳入但标注「⚠未验证独立性」 |

**单独保留：**
| 类型 | 识别特征 | 处理 |
|---|---|---|
| 品牌官网内容 | 品牌官方域下的「品牌故事」「品牌介绍」「关于我们」等 | ✅保留，备注「📌品牌官网发布」 |

> 去噪完成后，统计「原始搜索结果数」与「去噪后有效独立结果数」，以去噪后的数字作为可信度评级依据。

### Step 2：读取用户上传数据（可选）

用户可上传/粘贴以下任一数据，AI 自动提取关键信息：

- 小红书笔记导出（截图或文字）
- 品牌方提供的招商 PPT/手册
- 天猫/京东销售数据截图
- 第三方数据平台导出（5118/千瓜/蝉妈妈）
- 代理商同行反馈

处理方式：
- 用户粘贴文字 → 直接分析
- 用户发截图/文件 → 用 vision_analyze 提取数据
- 用户指定文件路径 → 用 read_file 读取

### Step 3：对标品牌分析（差距清单）

输出格式：

```
【对标品牌】XXX（行业标杆/直接竞品）

{品牌名} 不如 {对标品牌} 的地方：
1. 维度：具体差距（数据/事实支撑）
2. 维度：具体差距
3. ...

{品牌名} 优于 {对标品牌} 的地方：
1. 维度：具体优势
2. ...

差距总结（一句话）：{品牌名} 在 ___ 方面是对标的 ___%，在 ___ 方面是对标的 ___%
```

### Step 4：品牌代理模型分析 + 正负面痛点

按照品牌代理评估框架，逐项分析：

```
一、品牌底盘
- 成立时间/创始人/融资/团队规模
- 一句话定位/Slogan
- 品牌发展阶段（起步期/成长期/成熟期/衰退期）

二、产品线
- 品类结构 + SKU数量
- 核心爆品（Top 1-3，附价格）
- 价格带（入门→旗舰）
- 新品节奏

三、产品特性
- 核心成分/技术壁垒
- 差异化卖点（一句话说清"为什么选你"）
- 研发背书/认证

四、目标客户
- 官方宣称的目标人群
- 实际购买人群画像（年龄/性别/城市/消费力）
- 与代理区域用户重合度

五、痛点与需求
- 替代方案是什么？缺陷在哪？
- 品牌解决了什么功能需求？
- 品牌满足了什么情感需求？
- 「功能决定能不能卖，情感决定能不能卖上价」

六、品牌风格
- 视觉风格关键词（2-3个）
- 语言风格（文案调性）
- 包装/色系/质感

七、客户画像
- 典型用户的一天
- 她的消费观/价值观
- 她同时还在买什么品牌

八、正负面舆情摘要
- 正面 Top 3（附原文例句）
- 负面/风险 Top 3（附原文例句）
- 翻车事件（如有，时间+原因+后果）

九、地域迁移适配（原区域 → 代理区域）
   ⚠ 如果品牌原区域 = 代理区域（本土品牌），此项简化为「本土品牌，无地域迁移问题」

   从以下 7 个维度逐一对比，给出适配评分（高/中/低）和依据：

   a. 消费力匹配
      - 原区域客单价 vs 代理区域同品类客单价
      - 代理区域目标人群月均可支配收入 vs 品牌定价带
      - 结论：价格是否需要调整？提价空间还是降价压力？

   b. 审美偏好差异
      - 品牌视觉风格在原区域是否成立？（比如日系极简在欧洲可能被认作"性冷淡"）
      - 代理区域主流审美是什么？品牌风格是"降维"还是"水土不服"？
      - 包装/色系/质感在代理区域会不会被认作"廉价"或"用力过猛"？

   c. 消费习惯差异
      - 购买渠道偏好（原区域是线下为主还是线上？代理区域呢？）
      - 决策路径差异（原区域用户是被什么说服的？代理区域用户吃这一套吗？）
      - 使用场景差异（原区域的卖点在代理区域还成立吗？）

   d. 品类认知度
      - 这个品类在代理区域是"刚需"还是"教育市场"？
      - 代理区域用户对这个品类的主流认知是什么？跟品牌的核心卖点一致吗？
      - 是否需要先教育用户，还是可以直接收割？

   e. 文化适配
      - 品牌故事/价值观在原区域的文化土壤里成立，换到代理区域还成立吗？
      - 有没有文化冲突点？（宗教、习俗、政治敏感）
      - 品牌名/产品名在代理区域语言里有没有歧义或负面联想？

   f. 气候/地理适配
      - 产品是否依赖特定气候条件？（如护肤品在潮湿vs干燥地区的肤感差异）
      - 物流/仓储在代理区域是否可行？

   g. 竞品生态差异
      - 品牌在原区域的竞争地位 vs 代理区域的竞争格局
      - 代理区域已有类似定位的品牌吗？市场是空白还是红海？
      - 品牌是"第一个进来的"（先发优势）还是"后来者"（需要差异化）？

   综合结论：
   - 地域适配度：高 / 中 / 低
   - 一句话判断：
     「这个品牌从{原区域}搬到{代理区域}，最大的红利是___，最大的坑是___」

十、企业风险扫描
   数据来源：企查查 / 天眼查 / 启信宝公开信息。逐项列出条数和类别：

   ⚠️ **重要限制**：企查查(qcc.com)、天眼查(tianyancha.com)、爱企查(aiqicha.baidu.com)均部署了反爬/WAF机制，browser_navigate 在无 residential proxy + 无登录 Cookie 的环境下会被全部拦截（405 / CAPTCHA / 异常检测）。**无法通过浏览器自动化完成在线企业查询。** 如果百度百科词条包含企业统一社会信用代码、注册资本、法人等基础工商信息，可引用为「📌品牌官网发布」类别，但**不能替代独立第三方风险数据**。

   应对方案：
   - 标注风险等级为「⚠未完成（平台反爬拦截）」
   - 将「通过企查查/天眼查 APP 手动查询企业风险」作为**谈判报告的第一条重点核查事项**
   - 详见 `references/bing-search-pitfalls.md` 陷阱七

   | 风险类别 | 条数 | 详情摘要（如有） |
   |---------|------|----------------|
   | 司法诉讼 | X 条 | 原告/被告、案由、金额 |
   | 经营异常 | X 条 | 列入时间、原因、是否已移出 |
   | 行政处罚 | X 条 | 处罚机关、原因、金额 |
   | 股权冻结/质押 | X 条 | 涉及股东、金额 |
   | 失信被执行人 | X 条 | 案号、金额 |
   | 欠税/破产 | X 条 | 金额/状态 |
   | 其他（环保/消防/劳动仲裁等） | X 条 | 简述 |

   风险等级判定：
   - 0 条 → 🟢 干净
   - 1-5 条 → 🟡 有记录但可控，需在谈判中确认
   - 6-20 条 → 🟠 风险较高，建议重点核查
   - > 20 条 → 🔴 高风险，强烈建议暂缓合作
```

### Step 5：多维打分

| 维度 | 权重 | 评分标准 | 得分(1-10) |
|------|------|---------|-----------|
| **品牌力** | 20% | 知名度+口碑+品牌故事完整度 | |
| **产品力** | 25% | 差异化+技术壁垒+爆品潜力 | |
| **市场匹配度** | 25% | 代理区域需求+人群重合度+价格适配+地域迁移适配度 | |
| **增长潜力** | 15% | 品类趋势+品牌增速+市场空白 | |
| **合作风险** | 15% | 负面声量+翻车历史+竞品威胁+政策风险 | |

**总分 = 加权求和**

打分依据：
- 8-10 分：有明确数据支撑的强项
- 5-7 分：一般水平，有亮点但不突出
- 1-4 分：明显短板或风险信号

**Go/No-Go 判断：**
- ≥7.5 分 → 建议推进，报告中重点标注谈判筹码
- 5.5-7.4 分 → 有条件推进，报告中重点标注需压价的弱点
- <5.5 分 → 建议暂缓，除非对方有其他维度不可替代的价值

## 注意事项

1. **数据标注可信度**：每个数据点标注来源和可信度（⭐确定 / ◐估算 / ⚠推测）
2. **竞品差距要量化**：不是"XX不如YY"，是"XX的XX指标只有YY的XX%"
3. **打分要有依据**：每项打分附一句话理由，不要凭空给分
4. **报告可截图**：核心结论页（评分+筹码+压价点+地域迁移风险）控制在手机一屏内，方便老板截图
5. **地域迁移分析**：如果品牌原区域 ≠ 代理区域，地域适配性是核心谈判维度之一。品牌在原区域成功不代表能平移——消费力、审美、习惯、文化、气候、竞品生态六个层面都要过一遍。本土品牌则跳过此项。
6. **参考示例**：`references/examples.md` 包含修丽可和 MMLG 两份完整报告，可作为输出格式参考。
7. **搜索陷阱速查**：`references/bing-search-pitfalls.md` 记录了5个实测搜索失败模式及修复方案。遇到搜索异常时优先查阅。
8. **SEO封锁止损**：当品牌为知名品牌且连续2条不同关键词搜索返回几乎相同的前8-10条结果时，这是SEO封锁信号。立即停止搜索、接受⚠低可信度、基于百科+行业常识完成报告。不要在10+条关键词上反复获取相同结果。
9. **严禁子代理搜索**：不要用 `delegate_task` 执行搜索步骤。浏览器操作在子代理中累积超时（实测600s不够跑完5-6条搜索）。搜索必须由主代理直接执行。
10. **🚫 谈判话术禁忌词**：在"谈判筹码""压价点""谈判话术建议"中，禁止使用"原单/尾货/仿品/跟单""品控不行""你们卖不动了"等触发品牌方防御的表述。原则：用"用户觉得"不用"产品有"，把负面包装成"线下能做线上做不到"，用"我们"不用"你们"。

## 与其他 skill 的关系

- **brand**：品牌管理工作流总入口。brand-agency-intel 是品牌全流程的第二步，完成代理评估和谈判准备。全流程：brand-deep-dive（深扒现状）→ brand-agency-intel（代理评估）→ brand-3layer（三层分发）。
- **brand-deep-dive**：七维社媒舆情分析（声量/口碑/竞品/内容/用户/价格/时间轴），适合快速了解品牌在中文社媒的讨论情况。brand-agency-intel 在此基础上扩展为十维代理评估模型，增加了企业风险扫描、地域迁移适配、多维打分和谈判报告输出。两个 skill 均含 SEO 去噪层，仅统计剔除品牌干预后的独立第三方数据。如果只需要舆情快照，用 brand-deep-dive；如果需要完整的代理谈判评估，用 brand-agency-intel。
- **brand-3layer**：在本文输出的谈判报告基础上，进一步做三层分发——谈判层（筹码卡片化）、管理层（心理建设前置）、终端层（导购话术+AI高销脚本）。
