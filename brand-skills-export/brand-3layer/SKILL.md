# 品牌四维复用框架

## 一句话定位

品牌调研做一次，输出拆四层——谈判用、对方管理用、我方管理用、终端用。同一份品牌情报，四套对象四套话。

## 四维结构总览

| 层级 | 对象 | 前置内容 | 产出 | 来源 |
|------|------|---------|------|------|
| **①谈判层** | 你自己（品牌方谈判对手） | 品牌完整情报（十维模型 + 七维舆情） | 谈判筹码清单 + 压价点 + 话术建议 | brand-agency-intel |
| **②a 对方管理层** | 品牌管理者（合作方决策人） | 品牌深扒中对方不知道/不愿面对的信息 | 心理建设内容包（降低防御 + 拉齐预期） | brand-deep-dive 负面/风险维度 |
| **②b 我方管理层** | 你自己的团队（内部决策+执行层） | 品牌全维度情报 + 自身资源配置 | 内部对齐包（财务预期/品牌定位/团队培训/风险前置/一句话决策） | 前序全部情报 + 自身业务认知 |
| **③终端层** | 门店导购（你自己的人） | 产品卖点 + 竞品对比 + 用户真实评价 | 话术结构 + AI高销互动脚本 | 前两层交叉提炼 |

## 执行流程

### 前置条件：完成品牌情报收集

先跑 `brand-deep-dive`（基础七维舆情），再跑 `brand-agency-intel`（十维代理评估），两份报告齐了才能进三层分发。

### 第一层：谈判层 → 你的筹码

这一层的核心逻辑是：**不生成新信息，从已有情报中提炼可用的谈判武器。**

输入：brand-agency-intel 完整报告（十维模型 + 多维打分 + 谈判筹码/压价点）
输出：谈判话术精简版

格式：

```
【{品牌名}】谈判速查卡
━━━━━━━━━━━━━━━━━━━━━━━━

我的优势（3条，每条带数据支撑）：
1. {优势} — 数据：{具体数字/事实}
2. ...
3. ...

对方的软肋（3条，可在谈判中自然引出）：
1. {软肋} — 我的切入方式：{一句话}
2. ...
3. ...

价格锚点：
- 对方报价区间：{范围}
- 我的目标价：{数字}，理由：{简短}
- 谈判底线：{数字}

谈判关键句（可直接用）：
- 开场："{一句话建立平等关系}"
- 施压："{一句话点出对方痛点但不让对方难堪}"
- 收拢："{一句话促成共识}"
```

### 第二层：管理层 → 心理建设前置

这一层的核心逻辑是：**在正式谈之前，把对方管理者需要消化的事实先铺好，降低谈判时的防御反应。**

输入：brand-deep-dive 的负面/风险维度 + brand-agency-intel 的风险数据
输出：心理建设内容包

**不是攻击品牌，而是帮对方看到"他们自己可能没意识到的短板"——用关心的语气，不用批判的语气。**

格式：

```
【{品牌名}】管理层心理建设包
━━━━━━━━━━━━━━━━━━━━━━━━

1. 市场现实（让他看到天花板）
   "{品牌名}目前在小红书的声量主要集中在{关键词}，但{竞品}已经占据了{关键词}的心智。
   如果我们现在进场，需要正面竞争的难度是{数据}。"

2. 用户真实反馈（不是你的意见，是用户的嘴）
   "我调研了最近30天的用户评价，提到{体验摩擦点}的占比超过{X}%。
   这不是我说的问题，是已经买了你们产品的人在说的。"
   — 注意：{体验摩擦点}用"用户觉得"开头，不用"产品有"开头。
   比如：✅"用户觉得线上看图下单跟收到的质感对不上"
        ❌"线上图文不符导致退货率高"

3. 代理区域的特殊性（不是拒绝，是需要适配）
   "{代理区域}的用户对{品类}的认知跟{品牌原区域}不一样——
   {具体差异+数据}。所以原来的打法需要调整{具体建议}，
   这不是否定品牌，是让它在这里活下来。"

4. 拉齐预期的一句话
   "我希望我们合作的结果是：三年后{代理区域}的用户觉得{品牌名}就该在这。
   而不是卖了一季就没人记得。所以有些东西我们现在就得聊清楚。"
```

**语言原则：**
- 用"我们"不用"你们"——建立同盟感
- 每个判断跟一个数据——不是你在挑战他，是数据在说话
- 不否定品牌价值，只指出适配缺口——"这个品牌在{原区域}的{优势}是成立的，但在{代理区域}需要另一个版本"
- 结尾要给方向——不只说哪里不行，还要说怎么做行

**🚫 禁忌表达（容易让品牌方觉得你在攻击/暗示他们有问题）：**
- ❌ "淘宝原单" / "尾货" / "跟单" / "仿品" — 暗示品牌供应链泄露或假货泛滥
- ❌ "品控不行" / "质量不稳定" / "面料缩水" — 用用户的嘴说事实，不要用你的嘴下诊断
- ❌ "你们线上卖不动了" / "天猫数据在跌" — 除非有公开数据，别说"你在衰退"
- ✅ 正确姿势：把负面信息包装成"线上做不到但线下能做到的事"。
  不是说"你们线上有假货"，而是说"线上用户摸不到面料，线下可以解决这个信任问题"。
  不是说"你们品控有问题"，而是说"用户评价里提到退换货体验影响了复购决策"。

> ⚠️ 注意区分：这一层是给**对方管理者**看的。我方管理者需要另一套内容（见下方 ②b）。

### 第二层B：我方管理层 → 内部对齐

这一层的核心逻辑是：**在接触品牌方之前，先把内部团队拉齐——为什么做这个品牌、赚什么钱、投多少、风险在哪、跟现有业务怎么配合。** 不是给对方看的，是给自己人看的决策包。

输入：品牌全维度情报 + 自身业务认知（现有品牌矩阵、团队能力、资金状况）
输出：内部对齐包

格式：

```
【{品牌名}】我方管理层内部对齐包
━━━━━━━━━━━━━━━━━━━━━━━━

1. 为什么是它？（一句话让团队听懂）
   "{品牌名}给我们的是{一句话价值}。
   我们店里现在缺的不是{A}，是{B}。{品牌名}补的就是{B}。"

2. 财务预期（钱怎么赚、多久回本）
   - 首批拿货：建议 {SKU数量} 个SKU，资金占用控制在 {金额} 以内
   - 定价策略：吊牌价{是否折扣}（理由：{品牌方线上价格纪律/行业惯例}）
   - 毛利率：按 {折扣} 拿货，吊牌价 {价格带}，毛利率 {百分比}
   - 坪效预期：前 {N} 个月{低/平}（{原因}），第 {M} 个月起爬坡
   - 回本周期：保守估计 {月数}

3. 与现有品牌矩阵的关系（避免内部打架）
   - {品牌名} 定位：{一句话定位}
   - 现有品牌定位：[逐条列出]
   - 冲突点：{如果价格带/客群重叠，这里说清楚}
   - 差异点：{品牌名}吃的是「{心理/需求}」，跟现有品牌的「{心理/需求}」不冲突
   - 建议：{陈列区隔/导购分工/会员分层}

4. 团队需要知道的事（执行层面的关键差异）
   ① 导购策略不同：
     卖现有品牌你可以说{A}，卖{品牌名}你得会说{B}。培训必须前置。
   ② 考核指标不同：
     不要只看成交率。{品牌名}的{某个关键漏斗指标}可能比成交率更重要。
     比如"试穿加微率"、"新人进店率"。
   ③ 库存管理不同：
     {品牌名}的SKU节奏{快/慢/限量}，滞销款处理策略跟大众品牌不一样。
     建议：{预警机制}

5. 风险前置（坏消息先说）
   - 线上比价风险：...
   - 品牌方直营冲突：...
   - 品牌认知度低：前{N}个月自然客流少，依赖存量客户+导购主动推
   - 其他：...

6. 一句话决策（老板一眼看的）
   "{品牌名}不是我们店里最{赚钱/拉客/有调性}的牌子，
   但它可能是让我们从'卖衣服的'变成'{一句话愿景}'的那张牌。
   首年投入{金额}，赌的不是{品牌名}，赌的是{趋势/人群}。"
```

**跟 ②a 的区别：**

| | ②a 对方管理层 | ②b 我方管理层 |
|---|---|---|
| **目的** | 降低对方防御、拉齐预期 | 内部对齐、统一策略、资源到位 |
| **语气** | "我们一起"（同盟感） | "我们为什么做、怎么做、风险在哪"（决策感） |
| **核心问题** | "凭什么跟你合作？你的条件合理吗？" | "这个牌子能卖吗？投入多少？多久回本？" |
| **谁看** | 品牌方决策人 | 你自己的团队和老板 |

### 第三层：终端层 → 导购话术 + AI高销互动脚本

这一层的核心逻辑是：**把品牌调研结果翻译成一线的销售语言。导购不需要知道品牌融了多少钱，但需要知道客户最可能问的三个问题怎么答。**

输入：品牌十维模型 + 七维舆情 + 用户真实评价 + 竞品对比
输出：话术结构 + 互动脚本

格式：

```
【{品牌名}】终端话术结构
━━━━━━━━━━━━━━━━━━━━━━━━

📌 客户最可能问的 5 个问题 + 标准回答：

Q1: "{客户常用提问方式}"
A1: "{标准回答，30字以内}"
   深层逻辑：{用户问这个问题真正在意的点}

Q2: "{客户常用提问方式}"
A2: "{标准回答}"
   深层逻辑：{真正在意的点}

Q3/Q4/Q5: 同上

━━━━━━━━━━━━━━━━━━━━━━━━

📌 犹豫信号 → 促成话术对照表

| 客户说 | 说明 TA 在担心什么 | 标准回应 |
|--------|-------------------|---------|
| "我再看一下" | 有顾虑但不想说 | "{一句话追问}" |
| "有点贵" | 没看到价值 | "{一句话价值锚点}" |
| "X品牌好像更便宜" | 在比价 | "{一句话差异化}" |
| "我朋友买了说一般" | 社交压力 | "{一句话化解}" |
| "过段时间再说" | 不紧迫 | "{一句话制造紧迫}" |

━━━━━━━━━━━━━━━━━━━━━━━━

📌 互动脚本（让对话不尬、不生硬、不背稿）

开场（客户进店/浏览商品时）：
  "{一句话，不提品牌，不推销，建立自然对话}"
  → 原理：{为什么这句话好使}

推荐（客户在看某个产品时）：
  "{一句话，基于客户正在看的东西自然引出}"
  → 原理：{为什么不是硬推}

促成（客户有购买信号但犹豫时）：
  "{一句话，基于品牌的核心差异}"
  → 原理：{为什么不是施压}

离店（客户没买要走时）：
  "{一句话，留钩子不追人}"
  → 原理：{为什么比说"欢迎下次再来"强}

━━━━━━━━━━━━━━━━━━━━━━━━

📌 AI 高销增强（如果有录音/对话数据）

如果你后续上了录音工具或手动记录了对话，可以在这个框架上做增强：

每日推送模板：
  "今天你接待了 {X} 位客户，
   其中 {Y} 次触发'贵了'信号，平均停顿 {Z} 秒。
   建议下次直接问：'{标准回应}'。
   今天比你强的导购是 {名字}，她在处理'{某场景}'时的回复是：'{话术}'。"

此模板跟 brand-agency-intel 的教培销售教练逻辑一致，
只是信息来源从录音变成了品牌情报。
```

## 跟其他 brand skill 的关系

```
┌─────────────┐    ┌──────────────┐
│brand-deep-  │    │brand-agency- │
│dive         │    │intel         │
│七维舆情     │    │十维代理评估  │
└──────┬──────┘    └──────┬───────┘
       │                  │
       └──────┬───────────┘
              ▼
      ┌──────────────┐
      │ brand-3layer │  ← 这一层
      │ 四维分发     │
      └──┬───┬───┬───┘
         │   │   │
    ┌────┘   │   └────┬──────┐
    ▼        ▼        ▼      ▼
  谈判层  对方管理  我方管理  终端层
  (筹码)  (铺垫)   (对齐)  (话术)
```

**编排逻辑：**
- `brand-deep-dive` 先跑 → 拿到七维舆情（声量、口碑、竞品、内容、用户、价格、时间轴）
- `brand-agency-intel` 后跑 → 在七维基础上扩展成十维代理评估，输出谈判筹码
- `brand-3layer` 最后跑 → 把前两步的产出拆成四层，分发给不同对象

**一句话解释四者关系：**
brand-deep-dive 告诉你品牌"现在怎么样"，
brand-agency-intel 告诉你"值不值得代理、怎么谈"，
brand-3layer 把这两份情报翻译成——
"谈判桌上一句话、对方管理层一页纸、我方管理层一份决策包、导购手里一张卡"。

## 使用方式

当用户做了品牌调研之后说：
- "帮我拆三层" / "拆四层"
- "给管理者做心理建设的内容"
- "我方管理层" / "内部对齐" / "内部决策包"
- "帮我生成导购话术"
- "终端伙伴的话术结构"
- "AI高销互动脚本"
- "管理层铺垫内容"
- "这个品牌的四维输出"

触发后自动加载 brand-deep-dive 和 brand-agency-intel 的前序结果（如果还没有则提醒先跑），然后按四维模板生成输出。
—— ②a（对方管理层）和 ②b（我方管理层）默认都出，用户也可以指定只要某一侧。
